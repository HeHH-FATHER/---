## 二点五、离线抽卡管道（ODS→DWD→DWS→ADS · 每 5 分钟）

> 抽卡从实时链路迁至离线管道。生成器含保底机制（软74/硬90/50-50），MR 过滤五星后聚合入 MySQL。

### 架构

```
Gacha-Record-Generator (Java, 含保底)
    ↓ 生成逐抽记录 JSONL
preprocess_gacha.py → HDFS /data/gacha/ods/   (ODS)
    ↓
CleanGachaMR.java → HDFS /data/gacha/dwd/     (DWD 过滤五星)
    ↓
GachaAggMR.java → MySQL ads_gacha_offline     (DWS 聚合 + ADS 写入)
    ↓
数据质量页 (/analysis/quality) 监控批次
```

### 程序清单

| 层 | 程序 | 执行位置 |
|---|------|:--:|
| ODS | `scripts/preprocess_gacha.py` | master0 |
| DWD | `mr_jobs/.../CleanGachaMR.java` | master0 (YARN) |
| DWS | `mr_jobs/.../GachaAggMR.java` | master0 (YARN) |
| ADS | `scripts/load_gacha_to_mysql.py` | master0 |

### 管道命令


## 三、实时链路操作（2026-07-09 · Spark Streaming 版）

> ⚠️ **Python 消费端已全部替换为 Spark Streaming。旧 Python 代码备份在 `实时链路备份/共享消费端/`。**

### 架构

```
                    start_generators.sh                  start_spark_streaming.sh
                    /                 \                  /                        \
Character-Build-Generator (Java)   SatisfactionProducerApp    BuildStreamingConsumer   SatisfactionStreamingConsumer
    --count 10 --loop                 --loop --interval 3      60s窗口, TOP4             per-record ZADD+LIST
         ↓                                ↓                        ↓                        ↓
    Kafka build-v2                  Kafka satisfaction-v1      Spark Workers (slave1/2)   Spark Workers
         ↓                                ↓                        ↓                        ↓
    BuildStreamingConsumer          SatisfactionStreamingConsumer  Redis build:*           Redis satisfaction:*
         ↓                                ↓
    Redis build:recent (LTRIM20)    Redis rt:satisfaction:ranking (ZSET, 121角色)
    Redis build:hot_chars (TOP4)    Redis satisfaction:top (TOP6 JSON)
```

### 链路参数速查

| 参数 | 练度 (Build) | 满意度 (Satisfaction) |
|------|-------------|---------------------|
| 生成器 | `Character-Build-Generator` (Java `--loop`) | `SatisfactionProducerApp` (Java `--loop`) |
| Kafka Topic | `build-v2` | `satisfaction-v1` |
| 生成频率 | 每秒 10 条 | 每 3 秒 1 轮 |
| 消费端 | **Spark Streaming** (`CharacterBuildStreamingConsumer`) | **Spark Streaming** (`SatisfactionStreamingConsumer`) |
| Consumer Group | `spark-build-v2` | `spark-satisfaction-v2` |
| 窗口 | 60s 滑动 / 3s flush | per-batch (3s) |
| Redis Keys | `build:recent` (LTRIM20) `build:hot_chars` | `rt:satisfaction:ranking` `satisfaction:top` |
| 数据源 | `build_stats.json` | `satify_data.json` → `提瓦特数据/角色满意度排行.json` |
| Spark Master | `spark://master0:7077`（`--conf spark.cores.max=1`） | `spark://master0:7077`（`--conf spark.cores.max=1`） |
| 所需 jars | `jedis-2.9.0.jar`, `kafka-clients-2.1.0.jar`, `spark-streaming-kafka-0-10_2.11-2.4.0.jar`, `commons-pool2-2.4.2.jar` | 同左 |
| Batch 间隔 | **1 秒**（逐条 LPUSH+LTRIM，保证滚动） | 3 秒 |
| ⚠️ 重启注意 | 清 checkpoint + 换新 consumer group，否则回放历史消息卡死 | 同上 |

## 六、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| Spark Streaming 起不来 | spark-streaming-kafka jar 缺失 | 下载：`curl -sL -o /root/abyss-pipeline/jars/spark-streaming-kafka-0-10_2.11-2.4.0.jar 'https://repo1.maven.org/maven2/org/apache/spark/spark-streaming-kafka-0-10_2.11/2.4.0/spark-streaming-kafka-0-10_2.11-2.4.0.jar'` |
| `NoClassDefFoundError: LocationStrategies` | jar 不在 `--jars` 里 | 确认 spark-submit 带上 `--jars ...spark-streaming-kafka...jar` |
| `SatisfactionRecord not serializable` | 模型类未实现 Serializable | 重新编译 jar（已修复） |
| Python 消费端残留 | 旧脚本仍在运行 | `pkill -f realtime_consumer; pkill -f hot_char; pkill -f satisfaction_consumer` |
| Kafka 连不上 Middleware:9092 | Kafka 未启动 | `ssh Middleware '/root/kafka2/bin/kafka-server-start.sh -daemon /root/kafka2/config/server.properties'` |
| Kafka 启动失败 ZK timeout | ZK 未就绪 | 先启 ZK，等 5 秒再启 Kafka |
| 生成器 0 条记录 | version 中文过滤编码问题 | 去掉 `--version` 参数，或确认文件编码 UTF-8 |
| MySQL 拒绝连接 | 未授权远程访问 | `GRANT ALL ON *.* TO root@'%'` |
| HDFS put 失败 | NameNode 未启动或 safemode | `hdfs dfsadmin -safemode leave` |
| 生成器中文字符乱码 | shell 环境 LANG 非 UTF-8 | `export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8` |
| 大屏数据不滚动 | 生成器没加 --loop | 启动命令必须含 `--loop` 参数 |
| Spark UI 看不到作业 | 端口冲突（4040 被占→4041） | `http://master0:4040` 和 `http://master0:4041` |
