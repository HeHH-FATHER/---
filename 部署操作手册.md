## 二、离线链路操作（在 master0 执行）

### 第1步：编译 MapReduce

```bash
cd /root/abyss-pipeline/mr_jobs
mvn clean package -DskipTests
# 产出: target/abyss-mr-1.0-SNAPSHOT-jar-with-dependencies.jar
```

### 第2步：生成数据

```bash
cd /root/abyss-pipeline/Abyss-Record-Generator/Abyss-Record-Generator
mvn clean package -DskipTests
java -jar target/abyss-record-generator-1.0-SNAPSHOT-jar-with-dependencies.jar \
  --stats 提瓦特数据/深渊角色使用率.json \
  --out /root/abyss-pipeline/output/ \
  --users 200 --dirty-rate 0.05
```

### 第3步：ODS 预处理

```bash
cd /root/abyss-pipeline
python3 scripts/preprocess_abyss.py --input output/ --version v6.6
# 产出: HDFS /data/abyss/ods/abyss_v66.jsonl
```

### 第4步：DWD 清洗

```bash
hadoop jar mr_jobs/target/abyss-mr-1.0-SNAPSHOT-jar-with-dependencies.jar \
  com.neu.abyss.AbyssCleanMR \
  -libjars $JACKSON_JARS \
  /data/abyss/ods/abyss_v66.jsonl \
  /data/abyss/tmp_clean
# 产出: HDFS /data/abyss/dwd/dwd_char_detail/ + dwd_team_usage/
```

### 第5步：DWS 聚合

```bash
# 角色聚合
hadoop jar mr_jobs/target/abyss-mr-1.0-SNAPSHOT-jar-with-dependencies.jar \
  com.neu.abyss.AbyssAggMR \
  /data/abyss/dwd/dwd_char_detail/ \
  /data/abyss/dws/dws_char_summary/

# 配队共现
python3 scripts/compute_team_cooccur.py \
  /data/abyss/dwd/dwd_team_usage/ \
  /data/abyss/dws/dws_team_cooccur/
```

### 第6步：ADS 加载到 MySQL

```bash
python3 scripts/load_ads_to_mysql.py \
  --host 100.103.177.85 --port 3306 \
  --user root --password 123456 \
  --database abyss_db
```

### 第7步：注册数据管理菜单（在 Middleware）

```bash
mysql -u root -p123456 --default-character-set=utf8mb4 < /root/abyss-pipeline/scripts/ads_menu.sql
redis-cli FLUSHALL
```

### 第8步：灌入全量角色数据（ads_char_summary）

```bash
# 从 HDFS DWS 导出
hdfs dfs -cat /data/abyss/dws/dws_char_summary/part-r-00000 > /tmp/dws_char.csv

# 传到 Middleware
scp /tmp/dws_char.csv root@Middleware:/tmp/

# 在 Middleware 上灌入 MySQL
mysql -u root -p123456 --default-character-set=utf8mb4 abyss_db -e "
LOAD DATA LOCAL INFILE '/tmp/dws_char.csv'
INTO TABLE ads_char_summary
FIELDS TERMINATED BY '\t'
(version_name, char_name, star, own_count, use_count, total_users,
 own_rate, use_rate, avg_constellation, avg_level);"
```

## 三、实时链路操作

### 第1步：启动 Kafka Producer（在 Middleware）

```bash
cd /root/abyss-pipeline/data_generator

# 抽卡 Producer（后台，500条/秒）
nohup python3 gacha_kafka_producer.py --rate 500 > /tmp/gacha.log 2>&1 &

# 练度 Producer（后台，200条/秒）
nohup python3 build_producer.py kafka --rate 200 > /tmp/build.log 2>&1 &
```

### 第2步：启动实时数据留存（在 Middleware）

实时数据默认只在 Redis 中保留几秒（5秒滑动窗口过期）。
运行此脚本定期从 Redis 读取快照写入 MySQL rt_* 表，用于后续验证。

```bash
cd /root/abyss-pipeline

# 每 5 分钟执行一次快照（后台常驻）
nohup python3 scripts/snapshot_rt_to_mysql.py --loop 300 > /tmp/rt_snapshot.log 2>&1 &
```

验证：
```bash
# 等几分钟后查询
mysql -u root -p123456 --default-character-set=utf8mb4 abyss_db \
  -e "SELECT * FROM rt_gacha_result ORDER BY id DESC LIMIT 5"
mysql -u root -p123456 --default-character-set=utf8mb4 abyss_db \
  -e "SELECT * FROM rt_build_snapshot ORDER BY id DESC LIMIT 5"
```

> 数据闭环：离线聚合结果 vs 实时快照 → `verify_consistency.py` 交叉验证，证明两条链路数据一致性。

### 第3步：编译并提交 Spark Streaming（在 master0）

```bash
cd /root/abyss-pipeline/streaming_jobs
sbt assembly
# 产出: target/abyss-streaming-platform-1.0.jar

# 提交三个作业（三个终端窗口或后台）
# 实时链① 抽卡聚合（5秒窗口 → Redis gacha:*）
spark-submit --master yarn --deploy-mode client \
  --driver-memory 1g --executor-memory 2g \
  --jars /root/abyss-pipeline/jars/jedis-2.9.0.jar,/root/abyss-pipeline/jars/spark-streaming-kafka-0-10_2.11-2.4.0.jar \
  --class GachaAggregator \
  target/abyss-streaming-platform-1.0.jar &

# 实时链② 练度聚合（5秒窗口 → Redis build:*）
spark-submit --master yarn --deploy-mode client \
  --driver-memory 1g --executor-memory 2g \
  --jars /root/abyss-pipeline/jars/jedis-2.9.0.jar,/root/abyss-pipeline/jars/spark-streaming-kafka-0-10_2.11-2.4.0.jar \
  --class BuildAggregator \
  target/abyss-streaming-platform-1.0.jar &
```

> 注意：还需要把 D 盘的 `jars/` 传到 master0：`scp D:\neu2026bd\jars\*.jar root@master0:/root/abyss-pipeline/jars/`

## 七、常见问题

| 问题 | 原因 | 解决 |
|------|------|------|
| ZK 启动报错 | 单节点无法达到 quorum | 三台 ZK 都启动 |
| Kafka 连不上 | ZK 未就绪或 broker 未启动 | 先启 ZK 再启 Kafka |
| MR 作业找不到 Jackson | YARN 容器无 Jackson jar | 使用 `-libjars` 参数分发 |
| Python 脚本 SyntaxError | Python 版本太低 | 确认 python3 → 3.10 |
| MySQL 拒绝连接 | 未授权远程访问 | `GRANT ALL ON *.* TO root@'%'` |
| HDFS put 失败 | NameNode 未启动或 safemode | `hdfs dfsadmin -safemode leave` |
