## 第3页：项目背景与架构（何宏昊 · 1min）

### 背景
- 《原神》深境螺旋每期数十万玩家数据
- 提瓦特小助手 60+ 期统计、203 期卡池、122 角色练度
- 数据为聚合统计，缺个体分析与实时监控

### 系统架构
```
数据生成 → 离线管道(4层仓库) → 实时流(Kafka+Spark) → AI分析 → 大屏+分析页
```
6 节点集群（master0/slave1/slave2/backup/Middleware/vserver）

## 第6页：满意度数据链路（郭忠平 · 3min）

### 满意度生成器
- Java 应用，121 角色三维评分（综合/能力/外观）
- 3 秒循环 → Kafka `satisfaction-v1`

### Spark Streaming 消费端
```
Kafka satisfaction-v1 → SatisfactionStreamingConsumer(3s batch) → Redis
```
- ZSET: `rt:satisfaction:ranking`（121 角色实时排行）
- LIST: `rt:satisfaction:trend:*`（LTRIM 60 趋势）
- STRING: `satisfaction:top`（TOP6 快照，减少 API 请求）

### 大屏满意度面板
- TOP6 实时排行 + 三项分数 + 涨跌 + 迷你柱状图

## 第7页：练度实时链路（何宏昊 · 1min）

### 练度生成器
- 122 角色基底，sqrt 权重压缩，50 条/秒 → Kafka `build-v2`

### Spark Streaming 消费端
- 1s batch，60s 滑动窗口 → Redis
- lrem 去重 + lpush 保证唯一
- 每次重启新 consumer group 防 offset 冲突
- 新增 `constellation_dist` C0~C6 分布（实时命座分析）
