## 第3页：项目背景

### 为什么选这个方向？

- 《原神》深境螺旋每期数十万玩家提交数据
- 提瓦特小助手社区积累60+期深渊统计、203期卡池、122角色练度
- 数据为聚合统计，缺少个体粒度分析与AI智能监控

### 我们的方案

```
聚合统计 → 反推个体 → 四层数据仓库 → 实时流计算 → AI分析 → 可视化大屏
```

覆盖**离线批处理 + 实时流计算 + AI大模型 + 可视化**全链路

## 第6页：数据生成与离线管道

### 深渊管道（何宏昊）
```
提瓦特API统计 → Abyss-Record-Generator → ODS(HDFS) → AbyssCleanMR(10条规则) → AbyssAggMR → load_ads_to_mysql
```
- 200人模拟 → 187条通过 → 14635行DWD → 115角色DWS
- 10条清洗规则：星级/命座/等级/字段缺失/BOX空/重叠/约束/人数/UID/Teams

### 抽卡管道（侯逸丞）
```
卡池统计 → Gacha-Record-Generator(保底) → preprocess_gacha → CleanGachaMR → GachaAggMR → load_gacha_to_mysql
```
- 保底机制：软保底74抽/硬保底90抽/50-50
- 每5分钟5000抽50玩家循环，五星率~1.4%

## 第7页：实时流计算

### 练度链路（何宏昊）
```
CharacterBuildGenerator(50条/s) → Kafka build-v2 → Spark Streaming(1s batch, 60s窗口) → Redis build:hot_chars
```
- sqrt权重压缩算法（解决极端差距）
- lrem去重 → lpush → LTRIM 20
- Consumer group每次重启唯一化（防止offset冲突）

### 满意度链路（郭忠平）
```
SatisfactionProducerApp(3s/轮) → Kafka satisfaction-v1 → Spark Streaming(3s batch) → Redis
```
- ZSET: rt:satisfaction:ranking（121角色排行）
- LIST: rt:satisfaction:trend:*（LTRIM60趋势）
- STRING: satisfaction:top（TOP6快照）

## 第14页：关键技术难点及解决

| 难点 | 解决方案 |
|------|---------|
| Spark重启回放历史消息 | Consumer group每次用System.currentTimeMillis()唯一化 |
| 热门角色练度全平权 | sqrt权重压缩算法，50条/秒产出 |
| CDN图片403防盗链 | `<meta name="referrer" content="no-referrer">` |
| AI小模型编造数字 | 数据null清洗 + 低temperature + R1推理蒸馏模型 |
| Java 8不支持java.net.http | 改用Spring RestTemplate |
| 深渊管道set -e导致退出 | 全命令加`\|\| true`容错 |
| hotChars 40次单查SQL 12.8s | 3次批量查询优化 |
