## 二、个人负责模块

### 2.1 深渊离线数据管道

**数据生成器（Abyss-Record-Generator）**：Java 应用，基于 Jackson JSON 解析与伯努利采样，从聚合统计数据反推个体玩家深渊记录（角色 BOX + 战绩）。支持命令行参数（--users、--seed、--dirty-rate），预计算 3126 对合法配队组合，加权随机配队选择。

**四层数据仓库**：
- ODS（preprocess_abyss.py）：JSON 配对合并 → JSONL → HDFS
- DWD（AbyssCleanMR.java）：MapReduce Map-only 作业，10 条清洗规则（星级/命座/等级/字段缺失/BOX空/队伍重叠/角色约束/人数/UID/Teams），MultipleOutputs 多路输出
- DWS（AbyssAggMR.java）：自定义 VersionPartitioner，角色指标聚合 + 配队共现统计
- ADS（load_ads_to_mysql.py）：PyMySQL 写入 MySQL

200→187→14635→115→65，全链路验证通过。

### 2.2 练度实时流计算

**生成器（CharacterBuildGenerator.java）**：122 角色基底，sqrt 权重压缩算法，50 条/秒 → Kafka build-v2。

**Spark Streaming 消费端（CharacterBuildStreamingConsumer.java）**：1 秒 batch，60 秒滑动窗口 → Redis。核心优化：lrem 去重 + lpush 保证唯一、每次重启新 consumer group 防 offset 冲突、新增 constellation_dist 命座分布聚合。

## 三、关键技术难点及解决

| 难点 | 解决方案 |
|------|---------|
| Spark 重启回放历史消息 | Consumer group 每次用 System.currentTimeMillis() 唯一化 |
| 热门角色练度全平权 | sqrt 权重压缩，50 条/秒 |
| CDN 图片 403 防盗链 | `<meta name="referrer" content="no-referrer">` |
| AI 小模型编造数字 | 清洗 null 数据 + 低 temperature + R1 推理模型 |
| 深渊管道 set -e 退出 | `\|\| true` 容错 |
| hotChars 12.8s 超时 | 3 次批量查询替代 40 次单查 |
| Java 8 无 HttpClient | 改用 Spring RestTemplate |
| AI 响应 108 秒 | 精简上下文 + 降低 max_tokens |
