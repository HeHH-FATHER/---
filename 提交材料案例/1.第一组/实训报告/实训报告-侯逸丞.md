## 二、个人负责模块

### 2.1 抽卡生成器（Gacha-Record-Generator）

**功能**：基于提瓦特小助手 203 期卡池统计 JSON，反推个体玩家抽卡记录。

**核心实现**：
- 保底机制：软保底 74 抽 / 硬保底 90 抽 / 50-50 机制
- 角色命座分布（C0 63%/C2 20%/C6 6%）
- 专武-角色关联分配
- 版本时间区间内日期随机化
- 支持 Kafka 直发 + 本地 JSON 文件双输出
- 自带统计验证函数（对比生成结果与源统计偏差）

**参数配置**：--count 抽卡数 / --players 玩家数 / --kafka-bootstrap-servers Kafka地址 / --out-dir 输出目录

### 2.2 离线抽卡管道（ODS→DWD→DWS→ADS）

**ODS 层（preprocess_gacha.py）**：Python 脚本处理生成器产出的逐抽 JSONL 记录，上传至 HDFS `/data/gacha/ods/`。

**DWD 层（CleanGachaMR.java）**：Hadoop MapReduce 作业，过滤五星出货记录，丢弃 3★/4★ 武器和角色。经过清洗后保留五星出货数据。

**DWS 层（GachaAggMR.java）**：MapReduce 聚合作业，按 item_name 分组 COUNT，计算每个五星物品出现次数以及五星率。

**ADS 层（load_gacha_to_mysql.py）**：Python 脚本连接 MySQL，写入 `ads_gacha_offline` 表。每批次一条 __ALL__ 汇总行 + N 条物品明细行。

**循环模式**：run_gacha_loop.sh 每 5 分钟 / 5000 抽 / 50 玩家自动化运行，Redis 同步批次时间戳。

### 2.3 卡池维表设计

`dim_banner` 表录入 203 期卡池历史数据，关联物品名与 UP 版本，支持按版本/角色/武器类型多维度查询。

### 2.4 抽卡可视化页面

- 各版本卡池流水对比（柱状图）
- 角色抽取 TOP10（横向柱状图）
- UP 角色持有率 vs 总投入抽数（蝴蝶图）
- 卡池历史记录表格（分页 + 搜索）

### 2.5 抽卡数据质量监控

- ODS→DWD→DWS→ADS 管道流程可视化
- 批次明细表格（ODS 原始 / DWD 五星 / DWD 丢弃 / 清洗率 / 物品数 / 五星率）
- 4 条清洗规则卡片（星级过滤/物品聚合/批次汇总/时效性检查）
- 时效性检查（最近批次时间、五星率基线 0.4%~0.9%）

### 2.6 UP 物品权重配置

玛薇卡 79.77% / 洛恩 20.23%，焚曜千阳 62.05% / 灾悔 37.95%，反映真实卡池概率分布。

## 三、关键技术难点及解决

| 难点 | 解决方案 |
|------|---------|
| 保底机制 Java 实现 | 独立状态机：计数器追踪当前保底进度，74/90 阈值触发 + 50-50 判定 |
| 五星率验证 | 内含统计函数，生成结果与源统计偏差控制在 0.1% 内 |
| 循环管道稳定性 | run_gacha_loop.sh 容错设计，每批次独立 ID，异常不中断 |
| 管道时效监控 | Redis gacha:last_batch 时间戳同步 |
