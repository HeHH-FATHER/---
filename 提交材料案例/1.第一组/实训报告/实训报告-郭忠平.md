## 二、个人负责模块

### 2.1 满意度生成器（SatisfactionProducerApp）

**功能**：从提瓦特小助手"角色满意度排行"JSON 读取 121 角色基础数据，每 3 秒循环生成一轮满意度投票结果，写入 Kafka `satisfaction-v1` Topic。

**核心实现**：
- 基于真实 121 角色满意度的三维评分（综合/能力/外观）
- 微随机扰动模拟投票波动（±2 分内）
- `--loop --interval 3` 循环模式，3 秒一轮
- 中文文件名软链接方案：`satify_data.json` → `提瓦特数据/角色满意度排行.json`，避免 Shell 截断

### 2.2 Spark Streaming 消费端（SatisfactionStreamingConsumer）

**功能**：消费 Kafka `satisfaction-v1` 消息，3 秒 batch 间隔，逐条处理后写入 Redis。

**Redis 数据结构设计**：

| Redis Key | 类型 | 说明 |
|------|------|------|
| `rt:satisfaction:ranking` | ZSET | 121 角色实时满意度排行（ZADD 更新） |
| `rt:satisfaction:trend:*` | LIST | 每个角色的历史趋势（LTRIM 保留 60 条） |
| `satisfaction:top` | STRING | TOP6 快照 JSON（减少大屏 API 请求） |

**关键技术点**：
- ZADD 合并分数：旧分 × 权重 + 新分 × 权重，平滑波动
- LIST LTRIM 60：保留最近 60 条趋势数据用于迷你柱状图
- STRING JSON 快照：避免大屏每次请求都 ZREVRANGE 全量 121 角色

### 2.3 大屏满意度面板

**实时角色满意度排行面板**：
- TOP6 角色列表（排名 + 角色名 + 综合/能力/外观三项分数 + 涨跌指示）
- 每行右侧趋势迷你柱状图（最近 22 条记录）
- 5 秒轮询 `getSatisfaction()` API

**满意度 KPI**：
- 满意度最大波动：`satisList` 中绝对值 `delta` 最大的角色名 + 涨跌值

### 2.4 数据源管理

- 软链接方案：`ln -sf 提瓦特数据/角色满意度排行.json satify_data.json`，英文路径避免 Shell 中文截断
- 编码配置：`export LANG=en_US.UTF-8 LC_ALL=en_US.UTF-8`，确保 Java 生成器正确处理中文 JSON

## 三、关键技术难点及解决

| 难点 | 解决方案 |
|------|---------|
| 中文文件名 Shell 截断 | 创建英文软链接 `satify_data.json` |
| ZSET 瞬时波动大 | 新旧分数加权合并算法 |
| SatisfactionRecord 不序列化 | 模型类实现 Serializable，重新编译 |
| TOP6 JSON 更新不及时 | STRING 快照每 batch（3s）覆盖一次 |
| ability/look 字段缺失 | Kafka 消息读取字段映射修复 |
