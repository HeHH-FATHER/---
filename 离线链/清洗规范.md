# 离线链 — 数据清洗规范

> 对应 MR 作业：`AbyssCleanMR`
> 数据格式：每个用户 2 个 JSON 文件

---

## 一、数据格式

### 角色BOX（`{uid}_角色box.json`）

```json
{
  "uid": "100000042",
  "characters": [
    { "name": "班尼特",      "star": 4, "constellation": 4, "level": 80 },
    { "name": "玛薇卡",      "star": 5, "constellation": 2, "level": 90 }
  ]
}
```

| 字段 | 类型 | 约束 |
|------|------|------|
| `uid` | String | 非空，9位数字 |
| `characters` | Array | 非空，至少1个角色 |
| `characters[].name` | String | 非空，必须在角色字典中 |
| `characters[].star` | Int | 只能 **4** 或 **5** |
| `characters[].constellation` | Int | **0 ~ 6** |
| `characters[].level` | Int | **1 ~ 90** |

### 深渊战绩（`{uid}_深渊战绩.json`）

```json
{
  "uid": "100000042",
  "teams": [
    {
      "half": 1,
      "team_index": 17,
      "members": [
        { "name": "莱欧斯利", "star": 5 }
      ]
    },
    {
      "half": 2,
      "team_index": 85,
      "members": [
        { "name": "玛薇卡", "star": 5 },
        { "name": "枫原万叶", "star": 5 },
        { "name": "希诺宁", "star": 5 },
        { "name": "班尼特", "star": 4 }
      ]
    }
  ]
}
```

| 字段 | 类型 | 约束 |
|------|------|------|
| `uid` | String | 非空，**必须与 BOX 的 uid 一致** |
| `teams` | Array | **长度必须为 2**（上半+下半） |
| `teams[].half` | Int | 只能 **1**（上半）或 **2**（下半） |
| `teams[].team_index` | Int | > 0 |
| `teams[].members` | Array | **1~4 人** |
| `teams[].members[].name` | String | 非空 |
| `teams[].members[].star` | Int | 只能 **4** 或 **5** |

### 文件间一致性

| 规则 | 约束 |
|------|------|
| 上半 vs 下半 | `teams[0].members` 和 `teams[1].members` 的 `name` **不能有重叠** |
| BOX vs 战绩 | 战绩中所有 `members[].name` **必须全部出现在** 对应 BOX 的 `characters[].name` 中 |

---

## 二、清洗规则（8 条）

### 角色BOX 部分

| # | 规则 | 判定条件 | 脏数据类型 | MR 输出目录 |
|---|------|----------|-----------|-------------|
| 1 | 星级合法 | `star != 4 && star != 5` | `bad_star` | `/data/dirty/bad_star/` |
| 2 | 命座范围 | `constellation < 0 \|\| constellation > 6` | `bad_const` | `/data/dirty/bad_const/` |
| 3 | 等级范围 | `level < 1 \|\| level > 90` | `bad_level` | `/data/dirty/bad_level/` |
| 4 | 关键字段 | `name` 为空或 `null` | `missing_field` | `/data/dirty/missing_field/` |
| 5 | BOX 非空 | `characters` 数组为空 | `empty_box` | `/data/dirty/empty_box/` |

### 深渊战绩部分

| # | 规则 | 判定条件 | 脏数据类型 | MR 输出目录 |
|---|------|----------|-----------|-------------|
| 6 | 队伍不重叠 | 上半 `members[].name` ∩ 下半 `members[].name` ≠ ∅ | `overlap` | `/data/dirty/overlap/` |
| 7 | 角色约束 | 战绩中某 `name` 不在 BOX 的 `characters[].name` 中 | `missing_char` | `/data/dirty/missing_char/` |
| 8 | 队伍人数 | `members` 为空 或 `length > 4` | `bad_team_size` | `/data/dirty/bad_team_size/` |

### 文件级规则

| # | 规则 | 判定条件 | 脏数据类型 | MR 输出目录 |
|---|------|----------|-----------|-------------|
| 9 | uid 校验 | `uid` 为空 / BOX 与战绩 `uid` 不一致 / JSON 解析失败 | `bad_uid` | `/data/dirty/bad_uid/` |
| 10 | teams 校验 | `teams` 不为 2 个 / `half` 不为 1 或 2 | `bad_teams` | `/data/dirty/bad_teams/` |

## 三、MR Mapper 输出 Key 设计

```
C|{uid}  → 干净数据（Clean），Reducer 透传到 /data/clean/
X|{脏数据目录}/{uid}  → 脏数据，Reducer 路由到对应 dirty 子目录
```

### HDFS 输出目录结构

```
/data/clean/
    {uid}_角色box.json
    {uid}_深渊战绩.json
    ...

/data/dirty/
    bad_star/        ← 规则1
    bad_const/       ← 规则2
    bad_level/       ← 规则3
    missing_field/   ← 规则4
    empty_box/       ← 规则5
    overlap/         ← 规则6
    missing_char/    ← 规则7
    bad_team_size/   ← 规则8
    bad_uid/         ← 规则9
    bad_teams/       ← 规则10
```

## 四、计数器

| Counter 名 | 含义 |
|-----------|------|
| `TOTAL_RECORDS` | 总文件数（BOX + 战绩各算1条） |
| `CLEAN_PASSED` | 通过全部校验 |
| `DIRTY_BAD_STAR` | 星级异常 |
| `DIRTY_BAD_CONST` | 命座越界 |
| `DIRTY_BAD_LEVEL` | 等级越界 |
| `DIRTY_MISSING_FIELD` | 关键字段缺失 |
| `DIRTY_EMPTY_BOX` | 空BOX |
| `DIRTY_OVERLAP` | 队伍角色重叠 |
| `DIRTY_MISSING_CHAR` | 角色不在BOX中 |
| `DIRTY_BAD_TEAM_SIZE` | 队伍人数异常 |
| `DIRTY_BAD_UID` | uid异常 |
| `DIRTY_BAD_TEAMS` | teams结构异常 |
| `DIRTY_PARSE` | JSON解析失败 |
