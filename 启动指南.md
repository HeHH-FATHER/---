### 1.3 启动生成器

> **🟢 新版（7/9，当前使用）：一键启动**
> ```bash
> cd /root/abyss-pipeline && bash start_generators.sh
> ```
> 脚本内容：先 `pkill` 清旧进程 → 启动练度生成器（每秒10条 → `build-v2`）→ 启动满意度生成器（每3秒1轮 → `satisfaction-v1`，数据源 `satify_data.json` 是 `提瓦特数据/角色满意度排行.json` 的软链接）→ `disown` 托管。日志在 `/tmp/build_gen.log` 和 `/tmp/satisfaction.log`。
> 本地备份：`scripts/start_generators.sh`

### 1.4 启动 Spark Streaming 消费端（替换原 Python 消费端）

```bash
cd /root/abyss-pipeline && bash scripts/start_spark_streaming.sh
```

**这个脚本做了：**
- 停掉 Middleware 上旧的 Python 消费端
- 提交 2 个 Spark Streaming 作业（`local[2]` 模式在 master0 本地跑，不占 Spark Worker）：
  - `CharacterBuildStreamingConsumer`（batch=1s）：消费 `build-v2` → Redis `build:recent` + `build:hot_chars`
  - `SatisfactionStreamingConsumer`（batch=3s）：消费 `satisfaction-v1` → Redis `rt:satisfaction:*` + `satisfaction:top`
- 日志：`/tmp/spark_build.log` + `/tmp/spark_sat.log`

**所需 jars**（均在 `/root/abyss-pipeline/jars/`）：
| jar | 用途 |
|-----|------|
| `jedis-2.9.0.jar` | Redis 客户端 |
| `kafka-clients-2.1.0.jar` | Kafka 消费者 |
| `spark-streaming-kafka-0-10_2.11-2.4.0.jar` | Spark Kafka 集成 |
| `commons-pool2-2.4.2.jar` | Jedis 连接池依赖 |

## 五、关键配置（已锁定，不要改）

| 配置 | 值 | 说明 |
|------|-----|------|
| Build Topic | `build-v2` | 练度实时 Kafka topic |
| Build Consumer Group | `spark-build-v2` | Spark Streaming 消费 `build-v2` |
| Satisfaction Consumer Group | `spark-satisfaction-v1` | Spark Streaming 消费 `satisfaction-v1` |
| Build 生成频率 | 每秒 10 条 | 实时链路 |
| 满意度刷新 | 3 秒 | ZSET + LIST，6 角色 TOP |
| Build 滑动窗口 | 60s / 3s flush | 按角色聚合 TOP4 |
| 抽卡管道 | 5 分钟 / 5000 抽 | 离线管道 ODS→DWD→DWS→ADS |
| 大屏抽卡面板 | 5 分钟轮询 | 读 MySQL ads_gacha_offline |
| 抽卡保底机制 | 74 软/90 硬/50-50 | `preprocess_gacha.py` |
| 前端 Build 轮询 | 1 秒 | `_pollBuild()` |
| 前端 Hot 轮询 | 3 秒 | `_pollHotChars()` |
| 前端 满意度轮询 | 5 秒 | `_pollSatisfaction()` |
